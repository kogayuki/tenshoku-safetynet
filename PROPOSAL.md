# 転職セーフティネット.AI ｜ サービス企画案

## 1. サービス概要

| 項目 | 内容 |
|------|------|
| **サービス名** | 転職セーフティネット.AI |
| **一言で言うと** | 退職・転職時に使える日本の公的支援制度を、AIが個別に診断・案内してくれるWebサービス |
| **形態** | Webサイト（Next.js / Vercelホスティング） |

---

## 2. 背景・解決する課題

### 課題

- 退職・転職時に使える公的支援制度は **10〜20種類**存在するが、ほとんどの人が知らない
- 制度ごとに **所管省庁が異なり**（厚労省・国税庁・市区町村）情報が分散
- ハローワークに行っても **「自分の状況に合う制度の組み合わせ」までは教えてもらえない**
- 結果として、本来もらえる給付金（失業給付・再就職手当・教育訓練給付など）を **取り損ねる人**が多数

### 仮説

「制度データベース × AIによる個別診断」で、**自分の状況に合った制度の使い方とタイムライン**を5分で提示できれば、利用率向上に貢献できる

---

## 3. ターゲットユーザー

| 優先度 | セグメント | 想定状況 |
|--------|----------|---------|
| **第一** | 自己都合退職を検討中の30〜40代会社員 | 「辞める前に何をしておけばよかった？」を知りたい |
| **第二** | 退職直後で次の仕事を探している人 | 失業給付・健康保険・住民税の手続きで混乱中 |
| **第三** | 教育訓練給付でリスキリングしたい人 | どの制度が使えるか分からない |
| **第四** | 人事担当者・社労士見習い | 制度の概要把握のリファレンスとして |

---

## 4. 提供価値

### コアバリュー

> **「自分の状況を5問答えるだけで、使える制度の利用順序とスケジュールがわかる」**

### 副次バリュー

- 各制度の正式名称・所管省庁・公式サイトへのワンストップアクセス
- 個別質問に答えるAIチャット（24時間）
- マガジン的デザインで「読み物」として安心感のあるUX

### B. AIタイムライン診断
- 5問の対話フロー → Claude Sonnet 4.6 → 個別アドバイス
- 「いつ・何を・どの順番で」を時系列で提示

### C. AI相談チャット
- ストリーミング対応のチャットUI
- 制度Markdown を自動コンテキスト注入
- 断定的助言を禁じるシステムプロンプト設計

### D. SEO基盤
- `sitemap.xml` / `robots.txt` 自動生成
- OGPメタデータ・構造化データ
- 静的生成（SSG）による高速表示

### E. 記事置き場（Phase 2準備済）
- `content/articles/*.md` を置けば自動公開

## 6. 競合・差別化

| 競合 | 特徴 | 当サービスの優位点 |
|------|------|------------------|
| ハローワーク公式 | 情報網羅・公式 | 横断検索・個別診断ができない |
| 民間転職メディア | SEO記事多数 | エージェント送客が目的、中立性に課題 |
| 社労士サイト | 専門性高い | 個別相談前提、無料層が薄い |
| ChatGPT等 | 個別質問可能 | 日本の制度に特化していない、最新性に難 |

### 差別化ポイント
1. **日本の制度に特化** + **AI個別診断**の組み合わせ
2. **マガジン路線のデザイン**（明朝体 + 深い赤系アクセント）で信頼感
3. **送客ビジネスではない**中立スタンス
4. **Markdown駆動**で運用コストが低く、情報更新が容易

## 7. デザイン方針

- **フォント**: Shippori Mincho B1（明朝体ヘッドライン）+ Noto Sans JP
- **カラー**: 温かみのあるオフホワイト背景 + 深い赤系アクセント（#B85042）1色のみ
- **意図**: AI生成サイトにありがちな淡グレー・Interフォントを排除し、お金の話を扱うため「**静かな信頼感**」を演出

## 9. 運用方針・コスト

### 運用

- **コンテンツ更新**: 制度Markdown（9ファイル）を編集して `git push` だけで自動反映
- **法令改正対応**: 最低年1回、できれば四半期ごとに数値・条件を見直し
- **AI監査**: 月1回、AI出力が断定的助言になっていないかサンプル監査

### コスト

| 項目 | 想定 |
|------|------|
| Vercel | Hobbyプラン（無料） |
| Anthropic API | 月1,000利用で **$50〜100** |
| ドメイン | 年 ¥1,500前後 |
| **月次運用コスト** | **約 ¥10,000以下**（小規模時） |

---

## 10. 法的・倫理的配慮

- AI出力は **「一般的情報提供」に限定**、断定的助言を禁ずるsystem prompt設計
- 個別判断を求められたら **「ハローワーク・自治体窓口・専門家へ」** と案内
- 制度情報は執筆時点の一般情報である旨を明示
- 自動記事生成（Phase 2）は **必ず人間レビューを通す** 設計

## 12. 想定リスクと対策

| リスク | 対策 |
|--------|------|
| AIが事実誤認した助言を出す | system promptで断定禁止、データベース外情報は窓口案内、月次監査 |
| 法令改正で情報が古くなる | 四半期見直し、将来は差分監視自動化 |
| 検索流入が伸び悩む | 記事コンテンツ拡充、構造化データ強化、長尾KW狙い |
| Anthropic APIコスト急増 | 利用上限設定、レート制限、必要に応じてキャッシュ実装 |
| 「個別判断」を求められて答えてしまう | プロンプト調整 + 定期サンプル監査 |
